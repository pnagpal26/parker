<decisions_made>

- Use `<contact_captured>` tag (not client-side NLP) for early contact detection — same reliable pattern as `<lead_data>`, NLP on stream is fragile
- Partial `/api/lead` call does NOT write a FUB transcript note — conversation is too short to be useful. Only full lead writes the note.
- Fix 2 prefixes FUB note with `[Captured at lead collection]` so Garima knows it may not be the complete transcript
- Don't delete from localStorage after sendBeacon — can't confirm success, Fix 2 is the guaranteed baseline
- `saveTranscriptToStorage` preserves existing personId — personId arrives async from partial lead API and subsequent message saves shouldn't overwrite it with null
- `partialLeadFiredRef` pattern mirrors `leadCapturedRef` — prevents double-firing across messages

</decisions_made>
